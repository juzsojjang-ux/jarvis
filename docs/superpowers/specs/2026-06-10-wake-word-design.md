# 설계: 웨이크워드 + 연속대화 (실제 자비스 업그레이드 1단계)

날짜: 2026-06-10
상태: 사용자 승인됨

## 전체 로드맵 (사용자 합의)

"최대한 실제 자비스처럼" 업그레이드를 4단계로 분할. 각 단계는 독립된 스펙→계획→구현 사이클.

1. **웨이크워드 + 연속대화** ← 이 문서
2. 능동적 자비스 (아침 브리핑, 배터리 경고, 리마인더 알림, 부팅 인사)
3. 능력 확장 (실제 자비스가 하는 일 전부)
4. 음성·속도 튜닝 (지속 반복)

## 목표

키를 누르지 않고 "자비스"라고 부르면 응답한다. 답변 후 잠시 동안은
웨이크워드 없이 follow-up이 이어진다. 기존 PTT(오른쪽 Alt)는 백업으로 유지.

## 결정 사항 (사용자 선택)

- 마이크 운용: **상시 듣기 + PTT 유지** (처리 전부 로컬, 외부 전송 없음)
- 인식 방식: **VAD + 위스퍼 게이트** (Porcupine/openWakeWord 대신)
  - 이유: 한국어 발음 "자비스" 완벽 인식, 기존 mlx-whisper 재활용(추가
    학습·계정 불필요), "자비스, 몇 시야?" 한 호흡 호출+명령 지원
- follow-up 윈도우: 기본 8초 (설정 가능)

## 아키텍처

현재 마이크는 PTT를 누른 동안만 열린다. 이를 **상시 마이크 스트림 1개**로
바꾸고 두 소비자가 공유한다 (스트림 2개 금지 — 장치 충돌·배터리 낭비).

```
                  ┌─ PTT 경로(기존): Alt 누름~뗌 구간을 잘라 STT
마이크 상시 스트림 ─┤
                  └─ WakeListener(신규): silero VAD로 발화 구간 감지
                        ↓ 발화 종료 시
                     Whisper 변환 (로드된 모델 재사용)
                        ↓
                     "자비스"로 시작? ─ 아니오 → 즉시 폐기 (저장·로그 없음)
                        ↓ 예
                     웨이크워드 뒤 텍스트를 파이프라인에 직접 투입
                     (STT 단계 생략 — PTT보다 오히려 빠름)
```

### 컴포넌트

- **MicStream** (신규, `jarvis/audio/`): 상시 16kHz 모노 입력 스트림 +
  링버퍼. 구독자 모델 — PTT 캡처와 WakeListener가 같은 스트림을 읽는다.
  장치 변경/스트림 사망 시 자동 재시작.
- **WakeListener** (신규): VAD(silero, onnxruntime — CPU ~1%)로 발화
  시작/끝 검출, 발화 버퍼링(최대 30초 캡), 종료 시 Whisper 변환, 웨이크워드
  프리픽스 매칭, 명령 텍스트 추출.
- **Orchestrator 확장**: 텍스트 직접 투입 경로 `_on_wake_command(text)`
  (기존 `_pipeline`의 STT 이후 구간 재사용). follow-up 윈도우 관리.
  상태 게이팅(아래).

### 동작 규칙

- "자비스, 지금 몇 시야?" → 한 호흡에 바로 답변.
- "자비스"만 → "Yes, sir?" (자막 "네, 주인님?") 응답 후 follow-up 윈도우.
- **follow-up**: 답변 종료 후 8초간 모든 발화를 웨이크워드 없이 명령으로
  처리. HUD 링이 은은히 빛나 "듣는 중" 표시. 무발화 시 조용히 종료.
- **에코 억제**: 상태가 SPEAKING인 동안 + 종료 후 0.5초간 웨이크 감지
  일시정지(자기 목소리 자기인식 방지). 음성 barge-in은 에코캔슬링(AEC)이
  필요하므로 이번 단계 제외 — barge-in은 기존 PTT로.
- **PTT 공존**: PTT 누름은 진행 중 웨이크 처리를 취소하고 우선권을 가진다.

### 엣지 케이스 / 에러 처리

- 오인식: 변환 텍스트가 웨이크워드(자비스/쟈비스/jarvis, 구두점 무시)로
  *시작*해야만 반응. TV·일반 대화·위스퍼 환청("감사합니다" 류)은 폐기.
- 소음: VAD 문턱값 설정 가능. 발화 30초 캡(긴 대화 변환 낭비 방지).
- 장치 변경(이어폰 등): 스트림 에러 → 백오프 후 자동 재시작.
- 프라이버시: 전부 로컬. 비(非)자비스 발화는 변환 직후 메모리에서 폐기,
  로그·파일 기록 금지.

### 설정 추가 (config.py)

- `wake_enabled: bool = True`
- `wake_words: list[str] = ["자비스", "쟈비스", "jarvis"]`
- `follow_up_s: float = 8.0`
- `wake_vad_threshold: float = 0.5` (silero 발화 확률 문턱값)
- `wake_max_utterance_s: float = 30.0`

### 테스트

- 단위: 웨이크워드 프리픽스 매칭(변형·구두점), follow-up 윈도우 타이밍,
  SPEAKING 중 억제, PTT 우선권, 30초 캡, 빈/환청 발화 폐기.
  가짜 스트림/VAD/STT로 격리.
- 라이브: "자비스" 단독 호출, 호출+명령 한 호흡, follow-up 연속 질문,
  자비스 발화 중 자기반응 없음, PTT 백업 동작.
